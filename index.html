Setup Overview

This guide will walk you through the entire one-time setup process to get your Windows computer ready with Git and linked to GitHub. Please follow each step carefully. The goal is to install our tools and securely connect your machine to our shared workspace on GitHub.

5 Steps
15 Minutes
1 Time Setup
1

Install Git & Git Bash

Core version control engine with terminal

What this is: Git is the core "engine" for version control. This installer also includes Git Bash, the special terminal we will use for all our commands.

Action:

  1. Go to the official Git website: https://git-scm.com/download/win
  2. Download the installer and run it
  3. You will see many screens with lots of options. Do not change any settings. Just keep clicking "Next" until you see an "Install" button
  4. Click "Install", and then "Finish" when it's done
Pro Tip: The default settings are perfectly configured for most users. Stick with them!
2

Set Up Your GitHub Account & Choose Your Email Setting

Cloud hosting for shared code collaboration

What this is: GitHub is the cloud service where we will host our shared code. You need an account and must decide on an email setting.

Action:

  1. Go to GitHub.com and create an account. We recommend using your school email for a professional look
  2. Once logged in, you need to choose your email privacy setting
Option A (Recommended)
Recommended

If you used your public school email: Keep it showing. This looks professional and links your work directly to your academic identity. The only minor risk is potential spam.

Option B

If you used a personal email: Use a private email. This hides your real email for maximum privacy.

If you choose Option B, enable it in your settings:

  1. Click your profile picture (top-right) â†’ Settings
  2. In the left menu, click Emails
  3. Check both boxes: Keep my email addresses private and Block command line pushes that expose my personal email
  4. Copy the no-reply email address GitHub provides you
3

Configure Git Locally (Your Digital Signature)

Set up your identity for code commits

What this is: You need to tell the Git software on your computer the name and email to attach to every "commit" (save) you make. This is for attributionâ€”it's your signature on your work.

Action:

  1. Open Git Bash from your Start Menu
  2. Run the following two commands. For the email, use whichever one you chose in Step 2:
bash 
git config --global user.name "Your Full Name"
git config --global user.email "the-email-you-chose@example.com"
  1. Run this command to set the default branch name to the modern standard:
bash 
git config --global init.defaultBranch main
Important: If you enabled those two privacy options in Step 2, use the no-reply email. If you didn't, use the email you opened your GitHub account with.
4

Create & Link Your SSH Key (Your Secure Passkey)

Secure authentication without passwords

What this is: An SSH key is a secure passkey that your computer uses to prove its identity to GitHub. This is for authentication, and it allows you to push your code without ever needing to type a password.

Action:

  1. In Git Bash, run the following command to generate a key, using the same email you just configured:
bash 
ssh-keygen -t ed25519 -C "the-email-you-chose@example.com"
  1. The terminal will ask you three questions. For all three, just press Enter to accept the defaults
  2. Now, display your new public key and copy it to your clipboard by running this command:
bash 
cat ~/.ssh/id_ed25519.pub
  1. Highlight the entire block of text that appears, from ssh-ed25519 to the end, and copy it
  2. Go back to the GitHub website:
    • Go to Settings â†’ "SSH and GPG keys"
    • Click "New SSH key"
    • Give it a Title you'll recognize (e.g., "My Windows Laptop")
    • Paste your copied key into the "Key" text box
    • Click "Add SSH key"
Security Note: Never share your private key! Only share the public key (the one ending in .pub).
5

Verify Your Connection

Final check to ensure everything works

What this is: A final check to ensure everything is linked correctly.

Action:

  1. In Git Bash, run this final command:
bash 
ssh -T git@github.com
  1. You will likely see a warning about the authenticity of the host. This is a normal, one-time security question. Type yes and press Enter
  2. If everything worked, you will see a message that includes: Hi [your-username]! You've successfully authenticated...
Congratulations! Once you see that success message, your setup is complete. You are now ready to collaborate on the project.
